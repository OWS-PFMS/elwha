## §TL;DR — synthesis (read this first)

**What ElwhaSideSheet is:** the M3 **side sheet** — a surface containing supplementary content **anchored to the side of the screen**, in two types: **standard** (coexists with main UI on the same plane) and **modal** (blocks the UI behind a scrim). The generic side-panel primitive locked by #232 — filters, settings, detail panes, secondary navigation all compose onto it.

**Settled (canonical token set, §B):**
- **Width 256dp** (`docked.container.width`); modal max **400dp** (§E redline, MEDIUM confidence).
- **Standard:** container `surface`, elevation **level0**, shape **Corner.None** (square — it sits on the content plane).
- **Modal:** container `surface-container-low`, elevation token **level1** (⚠️ *not* a drop shadow — renders show the sheet flat over the scrim; §B correction), shape **Corner.Large (16dp)** on the **content-facing corners only** (MDC pairs the shape macro with `ShapeAppearanceOverlay…Corner.Left` for an end-docked sheet); **scrim 32%**.
- **Detached** (floating, 16dp margin, 16dp all corners) exists in the token set — **excluded from V1**, filed as the V2 epic (no silent cut).
- **Motion:** slide-translate 100% from the anchored edge, **275ms, emphasized easing, both directions** (MDC anim XML).
- **Sheet edge:** default **end/trailing**; left/right variants ship in MDC (RTL-aware).
- **Anatomy (modal):** back button (optional) · headline · container · close button · divider (optional) · actions (optional) · scrim. **Standard:** divider (optional, the sheet/content boundary) · headline · container · close affordance.

**Redline triangulation (not in the token set, §E):**
- Headline = **Title Large** — Google's own catalog content uses `textAppearanceTitleLarge` (HIGH).
- Content/footer horizontal padding **24dp**, header trails at 16dp next to the close button's 48dp target; footer = filled confirm + outlined dismiss, 12dp gap, divider above (MEDIUM — Flutter impl + spec-figure recall).
- Icon buttons 24dp glyphs in **48dp targets** (HIGH — M3 global a11y minimum, MDC doc repeats it).

**Elwha mapping: zero new theme tokens** (§F). Surface roles, `ShapeScale.LG` (16px), `SpaceScale`, `TypeRole.TITLE_LARGE`, `StateLayer`, scrim + 32% (dialog already does exactly this), `MorphAnimator` + `Easing.EMPHASIZED`, `ElwhaLayers.OVERLAY_LAYER`; **no `ShadowPainter`** — both types render flat (§B correction).

**Open questions → resolved in the design doc:** host architecture (one surface, two presentation paths), z-band (190), standard-mode reflow semantics (Swing-native ≈ M3 coplanar), 275→300ms adaptation, modal-width clamping. See `elwha-side-sheet-design.md` TL;DR for the locks.

## §A. Variants (MDC `SideSheet.md`)

| Variant | What it is | Android vehicle | Elwha disposition |
|---|---|---|---|
| **Standard** | Coexists with main UI; same plane; content visible alongside | `SideSheetBehavior` on a `CoordinatorLayout` child | **V1** [CODE] |
| **Modal** | Blocks the rest of the UI behind a scrim; Esc/scrim-dismissable | `SideSheetDialog` | **V1** [CODE] |
| **Coplanar** | A *standard* sub-variant that "squashes" a sibling view's width as the sheet expands (content reflows) | `app:coplanarSiblingViewId` | **V1-equivalent by construction** — a Swing-embedded standard sheet reflows siblings natively (see design doc §3). [CODE] |
| **Detached** | Floating sheet: 16dp margin all around, Corner.Large all four corners; exists for standard *and* modal | `Widget.Material3.SideSheet.Detached`, `…Modal.Detached` | **V2 epic** (filed — no silent cut) [CODE-deferred] |

- Drag/swipe semantics (`STATE_DRAGGING`/`STATE_SETTLING`, swipe-to-dismiss, `behavior_draggable`) are touch-first Android behaviors → **V2/DOC**; desktop V1 dismisses via Esc / scrim / close affordance / API.
- Predictive-back support is the Android back-gesture analog of our Esc handling → maps to Esc-to-dismiss [CODE], no separate surface.

⚠️ **Token-table ghost check:** the `md.comp.sheet.side` set contains *no* header/headline/typography/padding tokens at all — the header anatomy is redline-only (§E). Don't invent token names for it.

⚠️ **CORRECTION — modal elevation is not a drop shadow (operator screenshots, 2026-06-11 smoke loop):** the `container.elevation level1` token exists, but the m3.material.io renders (the modal-sheet-with-filters example and the modal anatomy redline) show the sheet **flat over its scrim** — no shadow halo anywhere; separation comes from the scrim contrast and the content-facing corner rounding. The elevation token is the *ghost* here: present in the set, expressed in no render. **Elwha paints both types flat** (no `ShadowPainter`, no shadow reserve, no `ShadowBearing`); the initial V1 build had derived a level-1 painted shadow from the token value and was corrected during smoke. The same renders also confirm the footer pairing: **filled confirm + outlined dismiss**, leading-aligned.

## §C. Anatomy (MDC `SideSheet.md`, numbered as the spec figures)

**Standard sheet:** 1. Divider (optional — the vertical boundary between sheet and main content) · 2. Headline · 3. Container · 4. Close affordance.

**Modal sheet:** 1. Back icon button (optional) · 2. Header · 3. Container · 4. Close icon button · 5. Divider (optional — above the action footer) · 6. Action(s) (optional) · 7. Scrim.

Notes:
- The standard sheet's divider is the **edge divider** (sheet/content boundary) — square-cornered, elevation-0 surface-on-surface needs the 1px line to read as a panel. [CODE]
- The modal sheet's divider is the **footer divider** (content/actions boundary). [CODE]
- Footer actions: confirm (filled) + dismiss (outlined/text) pattern in every observed render. [CODE, exact button variants MEDIUM §E]

## §D. Motion (MDC anim XMLs)

| Property | Value | Tag |
|---|---|---|
| Enter | translate `fromXDelta=100% → 0` (from the anchored edge), **275ms**, `motionEasingEmphasizedInterpolator` | [CODE] |
| Exit | translate `0 → 100%`, **275ms**, same emphasized easing | [CODE] |
| Left-edge variants | mirrored (`-100%`) | [CODE] |
| Scrim | fades with the transition (Android window dim); Compose/Elwha analog: fade scrim in sync with the slide | [CODE] |

- 275ms is an MDC hardcode sitting between M3 `motion.duration.medium1` (250) and `medium2` (300) — neither token is referenced. Elwha adaptation decision in the design doc §13 (lands on `MorphAnimator.MEDIUM2_MS` = 300, the lib-wide overlay duration). [CODE]
- Reduced motion: snap, no slide — `MorphAnimator` already auto-detects. [CODE]

## §H. Behavior & a11y capture

- **Modal:** scrim blocks interaction; tap-scrim dismisses (configurable); focus is trapped while open and restored on close; Esc dismisses. [CODE — all four already exist as `AbstractElwhaOverlay`/`AbstractElwhaDialog` semantics]
- **Standard:** no scrim, no focus trap; the sheet is part of the page's normal tab order; show/hide reflows content. [CODE]
- 48dp targets on the icon buttons; headline is the accessible name of the sheet. [CODE]
- Side sheets are for **supporting content/tasks related to the main view** (filters, detail panes); not for primary destinations (that's the rail) and not for interruptive confirmation (that's the dialog). [DOC]
- On compact windows a modal sheet may span the full width. [CODE: clamp to available width] / [DOC: guidance]

## §I. Open questions (answered — see design doc for the locks)

1. **One class or two?** One surface component, two presentation paths (embed = standard; `showModal()` = modal overlay). → design doc §2/§3.
2. **Which z-band for the modal sheet?** `OVERLAY_LAYER` 190 (below dialogs, above content) — pre-decided by #221 + menu doc §12. → §2.
3. **275ms vs lib-standard 300ms?** 300 (`MEDIUM2_MS`). → §13.
4. **Standard-mode reflow = coplanar?** Yes by construction in Swing. → §3.
5. **Detached + drag gestures?** V2 epic. → §16.
6. **Hard-clamp modal width at 400?** No — document the spec range, clamp only to the window. → §4.
